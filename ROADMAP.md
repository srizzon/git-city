# Git City Roadmap

This document tracks ideas and planned features for Git City, based on community feedback.

## 🏙️ City Planning & Zoning
- **Districts**: Implement logic to separate the grid into "neighborhoods" (e.g., Downtown, Suburbs).
- **Plot Selection**: Allow users to reserve or choose specific plots instead of random assignment.
- **Map Visualization**: Add a 2D minimap or overview to see empty vs. occupied plots.

## 🎮 Gamification & Progression
- **Achievements**: System to award badges for milestones (e.g., "500 Commits", "Early Adopter").
- **Leaderboards**: Highlight top contributors.
- **Easter Eggs**: Add fun elements like "Spider-Man" swinging between buildings.
- **Villains**: Represent bugs or issues as villains in the city.

## 💰 Economy & Monetization
- **Premium Plots**: Higher cost or requirements for "Downtown" plots.
- **Dynamic Pricing**: Plot value based on supply/demand of specific zones.
- **Visual Flair**: Premium banners, flags, or skins for buildings.

## 🛠️ Technical & UX
- **GitHub Auth**: Enforce GitHub login to build/claim a plot.
- **Time Travel**: Filter the city view by date (e.g., see the city as it was last year).
- **Performance**: Optimize rendering for high-density building areas.

## 📢 Social & Community
- **Project Promotion**: Allow users to link their buildings to their active projects.
- **Social Sharing**: Incentives for sharing city views on Twitter/TikTok.
